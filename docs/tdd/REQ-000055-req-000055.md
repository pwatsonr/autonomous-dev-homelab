## 1. Overview

This TDD wires the existing `autonomous-dev-homelab` design corpus (PRD-001, TDD-001/002, 6 plans, 29 specs, ~697 unit tests) to Patrick's live homelab. **The scope is integration and packaging, not design expansion.** No new fault probes, no new destructiveness categories, no new safety mechanisms вҖ” every live capability delivered here is a wiring of an already-designed subsystem to (a) real hosts, (b) a real secret store, and (c) a real installable artifact.

The design divides into eight workstreams, ordered by the critical path required to reach the PRD's Definition of Done:

1. **Build/Typecheck Repair (FR-BLD-01)** вҖ” targeted fix of `tests/cli/test-safety-cli.test.ts:133` `TS2493` without weakening `tsconfig.json`.
2. **Build Pipeline & Packaging (FR-BLD-02/03/04)** вҖ” new `emitDeclarationOnly=false` `tsconfig.build.json`, `scripts/build.sh`, `scripts/package.sh`, and a Claude Code plugin bundle artifact.
3. **Homelab Configuration (FR-CFG-01/02/03)** вҖ” a Zod-validated `homelab.config.yaml` schema declaring hosts, transport preference, and Vault references (never inline secrets).
4. **Vault-Backed Secrets (FR-SEC-01..04)** вҖ” a thin `SecretResolver` interface with an HashiCorp-Vault implementation (AppRole default), integrated at every credential boundary through a `CredentialRef` value type; redaction enforced at logger and audit boundaries.
5. **Live Discovery & MCP/SSH Connection (FR-DISC-*, FR-CONN-*)** вҖ” reuse `PlatformProber`, `MCPDiscovery`, `ConnectionPool`, `createConnection`; add a `homelab connect --test` command wrapping existing `connect-test` behind read-only probes; every attempt emits an audit record via the existing `AuditWriter`.
6. **Live Observation Loop (FR-OBS-*)** вҖ” bootstrap `ObservationCollector` with the SPEC-002-1-02 (Docker Swarm) and SPEC-002-1-03 (Unraid array/pool) probes and wire it into `homelab observe scan`.
7. **Safety-Gated Autofix (Dry-Run) (FR-SAF-*)** вҖ” no new safety code; the existing `safety/gate.ts` + `safety/delay.ts` + typed-CONFIRM enforce policy. This request adds a "propose only / dry-run" pathway and the `--abort-pending` CLI wired to `cancelDelayedAction`.
8. **Portal Panel, Deploy Scripts, Runbook, Evidence (FR-UI-*, FR-DEP-*)** вҖ” verify the existing portal panel renders live data, add `scripts/deploy/{install,upgrade,rollback}.sh`, write the operator runbook, capture evidence.

**Key design choices (see ADRs in В§14):**
- ADR-01: keep `tsc --strict`; fix `TS2493` via a tuple-narrowing helper, not by loosening compiler options.
- ADR-02: use HashiCorp Vault via `node-vault` (or thin HTTP client) with AppRole as documented default. **No Vault server or policy authoring in scope (PRD NG-03).**
- ADR-03: never persist resolved secrets вҖ” resolve on demand, hold in memory for the connection lifetime, hash-only in audit records (SHA-256 of the Vault path + secret version).
- ADR-04: **preserve the safety model as designed**. L0 actions against the live homelab are dry-run only for this request. `--abort-pending` uses the existing `cancelDelayedAction` sink.
- ADR-05: package as a Claude Code plugin using the existing `.claude-plugin/plugin.json` manifest; publish artifact as a versioned tarball; deploy scripts operate on `~/.claude/plugins/`.
- ADR-06: **MCP-first, SSH fallback per host, decided at connect time**, not at config time. Config declares both; runtime picks MCP if `MCPDiscovery` yielded a live endpoint, else SSH.

### 2.3 Data Flow

1. **Startup**: `runCli` calls `assembleRuntime(env, cwd)` вҶ’ loads `homelab.config.yaml`, constructs a `SecretResolver` (Vault via AppRole by default), assembles per-host `TransportProfile`s. If Vault is unreachable or config invalid вҶ’ fail closed, exit non-zero.
2. **Discover**: existing `runDiscover` extended with the config's declared hosts (skips `--cidr` if a config is present; iterates declared hosts). Each host is fingerprinted by `PlatformProber`. Inventory persisted via `InventoryManager` unchanged.
3. **Connect**: new `homelab connect --test` iterates declared hosts, calls `TransportSelect.pick(host)` вҶ’ returns `{transport: 'mcp'|'ssh', endpoint}`; each host is opened via `ConnectionPool.getConnection(id)` and closed after a single `ping()`-class read-only op. Every attempt writes an `audit.log` record.
4. **Observe**: `homelab observe scan` iterates connected hosts, invokes SPEC-002-1-02 (Swarm) or SPEC-002-1-03 (Unraid) probes via `live-probes.ts`, results flow into `ObservationCollector` вҶ’ `ObservationStore.persist()`.
5. **Autofix (dry-run)**: `homelab autofix propose` produces a `Proposal` JSON artifact under `.autonomous-dev/proposals/`. `homelab autofix dry-run <id>` executes safety validators (`safety/validator.ts`) and prints the ladder classification. **No mutation.**
6. **Portal**: existing `homelab portal` + `homelab metrics show` commands unchanged; verification is manual, screenshotted under `docs/evidence/`.

## 3. Detailed Design

### 3.1 Build & Typecheck Repair (FR-BLD-01)

**Problem**: `tests/cli/test-safety-cli.test.ts:133` вҖ” `audit.mock.calls[0]?.[0]` triggers `TS2493` under `strict` because `mock.calls[0]` is typed as a tuple with exactly `[event: CancelAuditEvent]` and TS infers `[0]` may be out of range in a general access context.

**Root cause**: The test's `jest.fn(async () => undefined)` is inferred without the `mock.calls` element type. When indexed with `[0]?.[0]`, the tuple index typing rejects positional access.

**Fix (ADR-01)**: Introduce a strongly-typed `jest.fn` cast at the call site вҖ” no change to `tsconfig.json`.

```ts
// tests/cli/test-safety-cli.test.ts (delta)
const audit: jest.MockedFunction<CancelAuditFn> = jest.fn(async () => undefined);
// audit.mock.calls[0][0] is now typed as CancelAuditEvent (no optional chain needed)
expect(audit.mock.calls[0][0]).toMatchObject({ type: 'action.cancelled', action_id: 'act-1' });
```

Where `CancelAuditFn` is already exported from `src/cli/commands/cancel-action.ts`. **Verification**: `npx tsc --noEmit` exits 0; the affected test still passes; no other files touched.

### 3.4 Configuration Schema (FR-CFG-01/02/03)

**File location**: `~/.autonomous-dev-homelab/homelab.config.yaml` (overridable via `--config <path>` or `AUTONOMOUS_DEV_HOMELAB_CONFIG`).

**Schema (Zod)** вҖ” `src/config/schema.ts`:

```ts
import { z } from 'zod';

const CredentialRef = z.object({
  vault_path: z.string().regex(/^[a-zA-Z0-9/_-]+$/),   // e.g. "kv/data/homelab/ssh"
  vault_field: z.string().min(1),                      // e.g. "private_key"
  // NEVER a `value` field. Static check + runtime rejection below.
}).strict();
export type CredentialRef = z.infer<typeof CredentialRef>;

const Host = z.object({
  hostname: z.string().min(1),
  platform: z.enum(['docker-swarm-manager', 'docker-swarm-worker', 'unraid']),
  role: z.enum(['manager', 'worker', 'nas']),
  mcp_endpoint: z.string().url().optional(),
  ssh_fallback: z.object({
    host: z.string().min(1),
    port: z.number().int().min(1).max(65535).default(22),
    user: z.string().min(1),
    key_ref: CredentialRef,
    known_hosts_ref: CredentialRef.optional(),          // pinned host key
  }),
}).strict();

export const HomelabConfig = z.object({
  version: z.literal(1),
  vault: z.object({
    address: z.string().url(),
    auth_method: z.enum(['approle', 'token', 'oidc', 'kubernetes']),
    approle: z.object({
      role_id_env: z.string(),
      secret_id_env: z.string(),
    }).optional(),
    // token/oidc/kubernetes sub-configs analogous
  }),
  hosts: z.array(Host).min(1),
```

### 3.5 Vault-Backed Secret Resolver (FR-SEC-01..04)

**Interface вҖ” `src/secrets/types.ts`**:

```ts
export interface ResolvedSecret {
  /** Raw material вҖ” held in memory only, never persisted. */
  readonly value: Buffer;
  /** For audit records вҖ” SHA-256("vault:" + path + ":" + field + ":" + version). */
  readonly refHash: string;
  /** Vault path + field, safe to log. */
  readonly ref: CredentialRef;
}

export interface SecretResolver {
  resolve(ref: CredentialRef): Promise<ResolvedSecret>;
  /** Health check for `connect --test` and startup validation. */
  ping(): Promise<void>;
}
```

**Vault implementation вҖ” `src/secrets/vault-resolver.ts`**:

```ts
export class VaultSecretResolver implements SecretResolver {
  constructor(private cfg: HomelabConfig['vault'], private env: NodeJS.ProcessEnv) {}

  async ping(): Promise<void> { /* GET /v1/sys/health, fail closed on non-2xx */ }

  async resolve(ref: CredentialRef): Promise<ResolvedSecret> {
    const token = await this.acquireToken();   // AppRole login вҶ’ client_token
    const resp = await this.kvRead(ref.vault_path, token);
    const field = resp.data?.[ref.vault_field];
    if (field === undefined) {
      throw new SecretMissingError(ref);       // fail closed, message = ref only
    }
    const value = Buffer.from(field, 'utf8');
    const version = resp.metadata?.version ?? 0;
    const refHash = sha256(`vault:${ref.vault_path}:${ref.vault_field}:${version}`);
    return { value, refHash, ref };
  }
}
```

**Redactor вҖ” `src/secrets/redactor.ts`**: attach as a sink filter to the existing `AuditWriter` and logger. Redaction strategy: (a) never accept a `Buffer`/`string` field named `value`, `secret`, `token`, `key`, `password` at the audit boundary; (b) whitelist of allowed audit fields per event type. Violations throw and the event is dropped with a `SECRET_LEAK_DETECTED` marker event containing only the field name.

### 3.6 MCP-First / SSH-Fallback Transport (FR-CONN-01)

**`src/live/transport-select.ts`**:

```ts
export interface TransportChoice {
  transport: 'mcp' | 'ssh';
  endpoint: string;
  reason: 'mcp-live' | 'mcp-unreachable' | 'mcp-not-configured';
}

export async function pickTransport(
  host: Host,
  mcpDiscovery: MCPDiscovery,
): Promise<TransportChoice> {
  if (host.mcp_endpoint !== undefined) {
    const alive = await mcpDiscovery.probe(host.mcp_endpoint, { timeoutMs: 2000 });
    if (alive) return { transport: 'mcp', endpoint: host.mcp_endpoint, reason: 'mcp-live' };
    return { transport: 'ssh', endpoint: host.ssh_fallback.host, reason: 'mcp-unreachable' };
  }
  return { transport: 'ssh', endpoint: host.ssh_fallback.host, reason: 'mcp-not-configured' };
}
```

Wired into a `LiveConnectionFactory` that adapts the existing `createConnection(id, entry)` API by extending `entry` with a resolved credential blob (in-memory, cleared after use). **`ssh-client.ts` is modified** to accept private-key bytes via `Buffer` rather than a filesystem path вҖ” no `~/.ssh` writes (FR-CONN-02).

### 3.7 Live Discovery (`homelab discover`) вҖ” FR-DISC-*

`runDiscover` is extended to take an optional `hosts: Host[]` from the config: when present, discovery iterates the declared hosts, calls `PlatformProber` against the declared endpoints (rather than a CIDR sweep), and asserts the observed fingerprint matches the declared `platform`. Mismatch вҶ’ error, exit 30 (`DISCOVERY_PLATFORM_MISMATCH`).

### 3.8 Live Connect (`homelab connect --test`) вҖ” FR-CONN-03/04

**New CLI вҖ” `src/cli/commands/connect.ts`**:

```ts
export interface ConnectDeps {
  config: HomelabConfig;
  resolver: SecretResolver;
  pool: ConnectionPool;
  mcpDiscovery: MCPDiscovery;
  audit: AuditWriter;
  streams: OutputStreams;
}

export async function runConnectTest(deps: ConnectDeps): Promise<number> {
  const results: PerHostResult[] = [];
  for (const host of deps.config.hosts) {
    const choice = await pickTransport(host, deps.mcpDiscovery);
    const start = Date.now();
    let outcome: 'ok' | 'fail' = 'ok';
    let error: string | undefined;
    try {
      const conn = await deps.pool.getConnection(host.hostname);
      await conn.ping();          // read-only op
      await conn.close();
    } catch (e) {
      outcome = 'fail';
      error = redact((e as Error).message);
    }
    const latencyMs = Date.now() - start;
    // Resolve credential ref hash without leaking the secret.
    const credRef = host.ssh_fallback.key_ref;
    const credHash = (await deps.resolver.resolve(credRef)).refHash;
    await deps.audit.write({
      type: 'connect.test',
      host: host.hostname,
      transport: choice.transport,
      transport_reason: choice.reason,
      outcome,
      latency_ms: latencyMs,
      credential_ref_hash: credHash,
      occurred_at: new Date().toISOString(),
      ...(error !== undefined ? { error } : {}),
    });
    results.push({ host: host.hostname, transport: choice.transport, outcome, latencyMs });
  }
  printTable(results, deps.streams);
  return results.every(r => r.outcome === 'ok') ? EXIT_OK : EXIT_CONNECT_FAIL;
}
```

## 8. Security Considerations

- **No plaintext secrets in repo/config/audit/logs/stdout/stderr**. Enforced by: (a) `.strict()` Zod schema, (b) `assertNoInlineSecrets` deep-walk guard, (c) `redactor.ts` audit-sink filter, (d) `scripts/secret-scan.sh` grep-based build guard, (e) unit tests asserting the redactor drops known-bad shapes.
- **Vault auth default is non-plaintext** (AppRole). Static token accepted for local test only; the runbook documents AppRole as the operator default (FR-SEC-02).
- **SSH host keys pinned via `known_hosts_ref`** where declared; missing pin surfaces a warning (`SSH_HOST_KEY_UNPINNED`). No auto-trust.
- **Credential lifetime**: `ResolvedSecret.value: Buffer` held in memory only for the duration of a `Connection`. On `close()` the buffer is zeroed (`buf.fill(0)`).
- **Audit HMAC**: existing behaviour, unchanged. Cancel/propose/dry-run events reuse the writer.
- **Ladder integrity (FR-SAF-01..03)**: no code path in this request can execute an L0 action live. The `MutationBarrier` wrapper is engaged in every dry-run pathway.
- **Threat model** вҖ” new attack surfaces introduced by this request:
  - *Malicious config* вҶ’ mitigated by strict Zod + inline-secret guard.
  - *Stolen AppRole SecretId in env* вҶ’ operator responsibility; documented in runbook.
  - *MITM on MCP endpoint* вҶ’ MCP transport already TLS-terminated (existing design); this request does not weaken that.
  - *Log siphoning* вҶ’ redactor drops secret-shaped fields; scan step confirms `dist/` clean.

## 14. Architecture Decision Records

### ADR-01: Fix `TS2493` without weakening `strict`

- **Context**: `strict: true` is a project invariant; the failing test asserts on a tuple index that strict typing rejects.
- **Options**:
  1. Loosen `tsconfig.json` (e.g., `noUncheckedIndexedAccess: false` вҖ” already false; adjust another flag) вҖ” **rejected**: PRD FR-BLD-01 forbids weakening strictness.
  2. Change the test to use `[0]?.[0]` with a non-null assertion вҖ” **rejected**: brittle.
  3. Type the mock explicitly via `jest.MockedFunction<CancelAuditFn>` вҖ” **chosen**: minimal, idiomatic, no config change.
- **Consequence**: The pattern becomes the reference for future jest-mock typing. Slightly more verbose test setup.
- **Traces**: PRD FR-BLD-01.

### ADR-02: Vault client via thin HTTP fetch (not `node-vault`)

- **Context**: We need Vault access for AppRole login + kv reads. `node-vault` is 300+ transitive deps.
- **Decision**: Implement a ~150-line HTTP client using Node's `fetch` targeting only the endpoints we use (`/v1/sys/health`, `/v1/auth/approle/login`, `/v1/{kv}/data/{path}`).
- **Consequence**: Minimal supply chain; we own the surface; must maintain when Vault API changes.
- **Traces**: FR-SEC-01..04, NFR-05.

### ADR-03: Never persist secrets; hash-only audit refs

- **Context**: PRD FR-SEC-01/04 requires zero secret material in any repo/log/audit artifact.
- **Decision**: `ResolvedSecret.value: Buffer` lives in memory only, zeroed on `Connection.close()`; audit records include `credential_ref_hash = sha256("vault:" + path + ":" + field + ":" + version)`.
- **Consequence**: Auditors can trace which credential was used without recovering the value; changing a Vault version rotates the hash.
- **Traces**: FR-SEC-01, FR-SEC-04, NFR-05.

### ADR-04: L0 dry-run only against the live homelab for this request

- **Context**: PRD FR-SAF-03 forbids executing L0 autofixes live; PRD G-04 requires demonstrating the loop.
- **Decision**: `autofix propose`/`dry-run` cover the demonstration path; `MutationBarrier` connection wrapper enforces read-only in dry-run.
- **Consequence**: End-to-end evidence includes a queued action + `abort-pending` before its 24 h delay elapses (R-06 mitigation).
- **Traces**: FR-SAF-01..04, G-04, G-05.

### ADR-05: Package as Claude Code plugin tarball; deploy via symlink flip

- **Context**: We need reversible install/upgrade/rollback (NFR-07).
- **Decision**: Store versioned installs at `~/.claude/plugins/autonomous-dev-homelab-<version>/`; `~/.claude/plugins/autonomous-dev-homelab` is a symlink. Rollback flips the symlink.
- **Consequence**: Rollback is atomic and cheap; retention window of 5 versions bounds disk use.
- **Traces**: FR-BLD-04, FR-DEP-01, NFR-07.

### ADR-06: MCP-first / SSH-fallback decided at connect time

- **Context**: MCP availability is dynamic; config-time selection would be stale.
- **Decision**: `pickTransport(host)` probes MCP at each connect attempt with a 2 s timeout; falls back to SSH otherwise. The choice + reason are recorded in the audit log.
- **Consequence**: Slight per-connect overhead (~2 s in the worst case); operators can reason about actual transport in place of guessing.
- **Traces**: FR-CONN-01, FR-CONN-04.
