## 1. Problem Statement

The `autonomous-dev-homelab` plugin has an extensive design corpus (1 PRD, 2 TDDs, 6 plans, 29 specs) and ~697 unit tests, but **nothing has been run against the live homelab**:

- The plugin has never connected to real hosts (Swarm manager, Swarm worker, Unraid Tower).
- The build path is broken: `tsc` fails at `tests/cli/test-safety-cli.test.ts:133` with a `TS2493` tuple-index error, which blocks `npm run build`, blocks producing `dist/cli/index.js`, and blocks producing an installable plugin artifact.
- There is no deploy / packaging path for the plugin; a homelab operator has no reproducible way to install it as a Claude Code plugin.
- Homelab connection configuration and secret retrieval (HashiCorp Vault) are unwired — the plugin cannot resolve credentials for real hosts and no plaintext credentials should exist anywhere on disk.
- The portal panel + SSE metrics for this plugin have never been rendered against the real portal.
- The observe → autofix loop has never executed end-to-end against a real host, so we have no evidence that the safety model (destructiveness ladder, typed-CONFIRM modal, 24-hour delay) survives contact with live traffic.

**Evidence:**

- `tests/cli/test-safety-cli.test.ts:133` — `TS2493` tuple index out of range, currently failing `npm run build`.
- No `build.sh`, packaging script, or `.claude-plugin` manifest that installs the plugin from a clean checkout.
- No integration/live-run artifacts exist under `docs/` or `.autonomous-dev/`.
- No runbook exists that a homelab operator could follow to go from a fresh clone to an installed, connected plugin.

**Why this matters now:** the design work is done, but the plugin is not usable. Until the live path works, none of the safety, observation, or autofix guarantees are proven — they are only asserted by unit tests.

## 6. Functional Requirements

Requirements are numbered `FR-<area>-<n>` and grouped. Every FR is testable.

### 6.1 Build & Package

- **FR-BLD-01** The `TS2493` error at `tests/cli/test-safety-cli.test.ts:133` MUST be resolved without weakening type strictness in `tsconfig.json`. **Verify:** `npx tsc --noEmit` exits 0.
- **FR-BLD-02** `npm run build` MUST produce `dist/cli/index.js` executable via `node dist/cli/index.js --help`. **Verify:** command exits 0; stdout is non-empty and matches a documented help fixture.
- **FR-BLD-03** A `scripts/build.sh` (or equivalent npm script) MUST exist that runs `npm ci`, typecheck, lint, unit tests, and build, in that order, and fails fast. **Verify:** invoking the script on a clean checkout completes green.
- **FR-BLD-04** A packaging step MUST emit an installable Claude Code plugin artifact (`.tgz` or plugin bundle) with a valid manifest. **Verify:** `claude plugin install <artifact>` (or documented equivalent) registers the plugin.

### 6.2 Configuration

- **FR-CFG-01** A config schema (JSON Schema or Zod) MUST validate the homelab configuration on load. Unknown fields, missing required fields, and malformed hostnames MUST produce actionable errors. **Verify:** unit tests cover accept/reject cases; live command exits non-zero on invalid config.
- **FR-CFG-02** Config MUST support declaring the three hosts in §7 with fields: `hostname`, `platform`, `role` (manager/worker/nas), `mcp_endpoint` (optional), `ssh_fallback` (host, port, user, key_ref).
- **FR-CFG-03** No secret material MAY appear in the config file. Secret fields MUST be references (Vault paths), not values. **Verify:** static check + runtime rejection of inline secrets.

### 6.3 Vault-Backed Secrets

- **FR-SEC-01** A Vault client MUST resolve secrets by reference at command time and never persist resolved values to disk. **Verify:** grep of all files written under `.autonomous-dev/` and `logs/` after a run contains no secret material.
- **FR-SEC-02** The Vault auth method (e.g., AppRole, token, OIDC) MUST be configurable and documented. Default MUST NOT be plaintext token in env.
- **FR-SEC-03** Vault unreachable, permission denied, or missing path MUST fail closed with a non-zero exit and a message identifying the auth method and Vault path (not the secret).
- **FR-SEC-04** Audit log entries MUST reference secrets by Vault path + hash of resolved reference, never the resolved value.

### 6.4 Discovery & Inventory (Live)

- **FR-DISC-01** `discover` MUST detect `gallifrey-lab-01` as `docker-swarm-manager`, `gallifrey-lab-02` as `docker-swarm-worker`, and `unraid.pwatson.space` as `unraid`. **Verify:** persisted inventory JSON matches expected fingerprints.
- **FR-DISC-02** Service enumeration MUST include Portainer, Sonarr, Radarr, SABnzbd, Authentik, NPM, Postgres, and OpenSearch on the Swarm cluster, and Plex + array + autofan on Tower. Missing services MUST be reported, not silently omitted.
- **FR-DISC-03** Discovery MUST be idempotent: two consecutive runs against unchanged infrastructure produce byte-identical inventory (excluding timestamps).

### 6.5 Connection Layer (Live)

- **FR-CONN-01** Connections MUST attempt MCP first and fall back to SSH only when MCP is unreachable or unsupported for the host. The transport used MUST be recorded in the audit log per attempt.
- **FR-CONN-02** SSH fallback MUST use key material referenced by Vault path, resolved at connection time. No `~/.ssh` writes.
- **FR-CONN-03** `connect --test` MUST perform only read-only operations against each host and MUST exit non-zero if any host fails. Per-host results MUST be listed in stdout.
- **FR-CONN-04** All connection attempts (success or failure) MUST emit audit-log records containing: `timestamp`, `host`, `transport`, `outcome`, `latency_ms`, `credential_ref_hash`.

### 6.6 Observation Loop (Live)

- **FR-OBS-01** `observe` MUST execute at least the container-health probes for Docker Swarm and the array/pool probes for Unraid, per SPEC-002-1-02 and SPEC-002-1-03.
- **FR-OBS-02** Every probe MUST persist a well-formed observation validated against the SPEC-002-1-01 schema.
- **FR-OBS-03** Probe execution MUST NOT mutate any live host.

### 6.7 Autofix Safety (Live)

- **FR-SAF-01** Every candidate autofix MUST be evaluated against the destructiveness ladder (SPEC-002-2-01). Destructive or live-mutating actions default to L0 and MUST require typed-CONFIRM and a 24h queue delay before execution.
- **FR-SAF-02** The typed-CONFIRM modal (SPEC-002-2-02) MUST persist an HMAC-signed record. Autofix execution MUST refuse to run if the HMAC is missing/invalid or the 24h delay is unmet.
- **FR-SAF-03** For this request, no autofix classified as L0 SHALL be executed against the live homelab. Only dry-run for L0; safe read-only or L2+ actions may execute if any exist.
- **FR-SAF-04** An `--abort-pending <id>` command MUST cancel a queued L0 action and log the abort.

### 6.8 Portal Panel + SSE

- **FR-UI-01** The homelab panel MUST render inventory, latest observations, and current safety-ladder queue against the live portal.
- **FR-UI-02** SSE metric updates MUST reach the panel within 5 seconds P95 of emission.
- **FR-UI-03** Panel errors (auth, transport, schema) MUST surface a visible banner rather than a blank panel.

### 6.9 Deploy, Runbook, Evidence

- **FR-DEP-01** A `scripts/deploy/install.sh` MUST install the built plugin into Claude Code. `upgrade.sh` and `rollback.sh` MUST provide reversible operations.
- **FR-DEP-02** An operator runbook MUST live at `docs/runbooks/REQ-000055-homelab-operator.md` covering: prerequisites, install, configure, Vault auth, discover, connect --test, observe, dry-run autofix, panel verification, rollback.
- **FR-DEP-03** Evidence artifacts (command transcripts, audit-log excerpts with redacted secret refs, panel screenshots) MUST be committed under `docs/evidence/REQ-000055/`.

### 6.10 Testing

- **FR-TST-01** All existing unit tests MUST remain green. Baseline: ~697 tests currently expected to pass once FR-BLD-01 is resolved.
- **FR-TST-02** Add integration/live-smoke tests behind a `LIVE=1` flag; default `npm test` MUST NOT touch live hosts.
- **FR-TST-03** Add a CI job (or documented local script) that runs typecheck, lint, unit tests, and build on every PR.
