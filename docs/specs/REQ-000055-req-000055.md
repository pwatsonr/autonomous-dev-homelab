## 0. Introduction

This spec translates the 16 plan tasks (TASK-001 … TASK-016) into implementation-ready contracts. It is intended to be read by executor agents alongside the parent TDD (which is the source of truth for the design corpus already produced) and the parent Plan (for task ordering, dependencies, and acceptance criteria). Where the plan or TDD already provides a bit-exact contract, this spec references it rather than duplicating it; where the plan requires elaboration to be implementable without further clarification (type signatures, error taxonomy details, sample fixtures, exact test cases), this spec provides it.

**Design rule:** Rigor is scaled to task complexity. Where a task introduces public APIs, persisted data, secret handling, or safety-relevant logic (TASK-004, TASK-005a, TASK-005b, TASK-006, TASK-007, TASK-008, TASK-009, TASK-010, TASK-011), this spec defines the full contract, schemas, errors, and concrete test cases. Where a task is a script or documentation artifact (TASK-002, TASK-003, TASK-012a, TASK-012b, TASK-013, TASK-014, TASK-016), behavioral acceptance criteria and shell command contracts are used instead of manufactured byte-level assertions.

**Universal invariants** (apply to every task):

1. Preserve `tsconfig.json` `strict: true` (ADR-01).
2. Preserve all ~697 existing tests green at every task boundary.
3. Default `npm test` MUST NOT open network connections to `gallifrey-lab-01`, `gallifrey-lab-02`, or `unraid.pwatson.space`. Live tests are gated behind `LIVE=1` env var.
4. Never persist a resolved secret. Only `refHash` (SHA-256 of `vault:{path}:{field}:{version}`) appears in any log, audit record, or evidence file.
5. Every new error class exposes stable `{ code: string, exit: number, message: string, details?: Record<string, unknown> }` and its stringification is redacted by `src/secrets/redactor.ts`.
6. All new CLI commands are wired via `src/cli/index.ts`; existing commands are not renamed or removed.
7. All new files are formatted per the repo's existing ESLint config; new tests use Jest (matching the existing test suite).

### 2.2 `src/config/schema.ts` (TASK-004)

Exact schema per TDD §3.4, restated here as the contract:

```ts
import { z } from 'zod';

export const CredentialRef = z.object({
  vault_path: z.string().regex(/^[a-zA-Z0-9/_-]+$/, 'vault_path must match /^[a-zA-Z0-9/_-]+$/'),
  vault_field: z.string().min(1),
}).strict();

export const Host = z.object({
  hostname: z.string().min(1),
  platform: z.enum(['docker-swarm-manager', 'docker-swarm-worker', 'unraid']),
  role: z.enum(['manager', 'worker', 'nas']),
  mcp_endpoint: z.string().url().optional(),
  ssh_fallback: z.object({
    host: z.string().min(1),
    port: z.number().int().min(1).max(65535).default(22),
    user: z.string().min(1),
    key_ref: CredentialRef,
    known_hosts_ref: CredentialRef.optional(),
  }).strict(),
}).strict();

export const HomelabConfig = z.object({
  version: z.literal(1),
  vault: z.object({
    address: z.string().url(),
    auth_method: z.enum(['approle', 'token', 'oidc', 'kubernetes']),
    approle: z.object({
      role_id_env: z.string().min(1),
      secret_id_env: z.string().min(1),
    }).optional(),
  }).strict(),
  hosts: z.array(Host).min(1),
}).strict();
```

**Invariants**:
- Every object is `.strict()` (unknown keys rejected).
- `CredentialRef` MUST NOT accept a `value`, `secret`, `token`, `password`, or `key` field. `.strict()` enforces this at parse time.
- When `auth_method === 'approle'`, `approle` MUST be present. Enforce via `.superRefine(...)`; on violation, path is `vault.approle`, message: `"approle sub-config required when auth_method is 'approle'"`.

```ts
export interface LoaderOptions {

 * @throws ConfigInvalidError (exit 11)  — YAML parse error or Zod violation or inline-secret

export function loadHomelabConfig(opts?: LoaderOptions): Promise<HomelabConfigT>;

/**
 * Deep-walk a parsed config and reject any string leaf whose key is in
 * DISALLOWED_LEAF_KEYS. Runs post-Zod so already-validated CredentialRef
 * shapes still trigger on any surviving disallowed key.
 * @throws ConfigInvalidError with `code: 'INLINE_SECRET'`
 */
export function assertNoInlineSecrets(cfg: unknown): void;

export const DISALLOWED_LEAF_KEYS: ReadonlySet<string>;
```

**`DISALLOWED_LEAF_KEYS`** MUST equal `new Set(['value', 'password', 'secret', 'token', 'private_key', 'privateKey', 'passphrase'])`. `key` is intentionally NOT in this set because the schema field `key_ref` is legitimate; leaf detection is triggered only on string values, not on subtree names.

**`formatZodError(err, path)`** MUST produce lines of the form:
```
config.hosts[1].ssh_fallback.key_ref.vault_path: <constraint message>, received <safe-repr(value)>
```
where `safe-repr` truncates strings > 40 chars, and NEVER renders values from paths ending in `.vault_field`, `.role_id_env`, or `.secret_id_env` (prints `<redacted>` instead).

**Precedence for path resolution**:
1. `opts.path` if provided.
2. `opts.env?.AUTONOMOUS_DEV_HOMELAB_CONFIG` if set.
3. `${os.homedir()}/.autonomous-dev-homelab/homelab.config.yaml`.

### 2.5 `src/secrets/vault-resolver.ts` (TASK-005a)

```ts
import type { VaultConfig } from '../config/types.js';
import type { CredentialRef } from '../config/types.js';
import type { ResolvedSecret, SecretResolver } from './types.js';

export interface VaultResolverOptions {
  /** Override fetch (for testing). Signature matches global fetch. */
  fetchImpl?: typeof fetch;
  /** Default 5000 ms. */
  requestTimeoutMs?: number;
  /** Token refresh at (leaseDuration * ratio). Default 0.9. */
  tokenRefreshRatio?: number;
  /** Clock (for testing). Default () => Date.now(). */
  now?: () => number;
}

export class VaultSecretResolver implements SecretResolver {
  constructor(cfg: VaultConfig, env: NodeJS.ProcessEnv, opts?: VaultResolverOptions);
  resolve(ref: CredentialRef): Promise<ResolvedSecret>;
  ping(): Promise<void>;
  /** Test/lifecycle hook. Cancels in-flight AbortController and zeroes cached token. */
  dispose(): void;
}
```

**HTTP endpoints** (used by `VaultSecretResolver`; documented so executors do not invent alternatives):

| Method | Path | Purpose |
|---|---|---|
| POST | `/v1/auth/approle/login` | AppRole login. Body: `{ role_id, secret_id }`. |
| GET | `/v1/sys/health` | Health probe used by `ping()`. Treat 200, 429, and 473 as reachable; 501, 503-with-standby as reachable-but-standby (returns void). |
| GET | `/v1/{vault_path}` | KV read. Server response shape: `{ data: { data: Record<string,string>, metadata: { version: number } } }` (KV v2). The `vault_path` MUST already include `/data/` for KV v2 mounts — the resolver does NOT rewrite it. |

**Token lifecycle**:
- After `POST /v1/auth/approle/login` returns `{ auth: { client_token, lease_duration } }`, cache the token with `expiresAt = now() + lease_duration * tokenRefreshRatio * 1000`.
- Subsequent `resolve` calls reuse the token if `now() < expiresAt`; else re-login.

**Compute `refHash`**:
```
sha256:${hex(sha256(`vault:${ref.vault_path}:${ref.vault_field}:${version}`))}
```
- `version` is `metadata.version` when present, else `0`.
- MUST be deterministic: same inputs → same hex.

**Error mapping** (see §3):

| HTTP status | Endpoint | Thrown error |
|---|---|---|
| Any connect refuse / DNS error / timeout | any | `VaultUnreachableError` |
| 400 / 403 / 400-invalid-role | `/v1/auth/approle/login` | `VaultAuthError` |
| 403 | `/v1/{path}` | `VaultPermissionError` |
| 404 | `/v1/{path}` OR field missing in response.data.data | `SecretMissingError` |
| 5xx (not standby) | any | `VaultUnreachableError` |

All resolver operations use an `AbortController` with `requestTimeoutMs` (default 5000).

### 2.6 `src/secrets/redactor.ts` (TASK-005b)

```ts
/** Recursively strip disallowed fields from an audit event payload. */
export function redactAuditEvent(event: Record<string, unknown>): Record<string, unknown>;

/** Redact a message string for user output; replaces long hex/token-like runs and known secret-ish substrings. */
export function redact(message: string): string;

/** Field names that MUST NOT appear in any audit event value position. */
export const REDACTED_KEYS: ReadonlySet<string>;

/** Attach to an AuditWriter. Wraps writer.write to run redactAuditEvent first;
 * emits SECRET_LEAK_DETECTED marker event if a REDACTED_KEYS hit is found. */
export function installRedactorSink<W extends { write: (e: Record<string, unknown>) => Promise<void> }>(writer: W): W;
```

**`REDACTED_KEYS`** = `new Set(['value', 'secret', 'token', 'password', 'key', 'private_key', 'privateKey', 'passphrase', 'client_token', 'role_id', 'secret_id'])`.

**Behavior**:
- On hit: replace value with `'<redacted>'`, emit a sibling marker event `{ type: 'SECRET_LEAK_DETECTED', field: <fieldName>, at: <ISO timestamp> }` through the same writer.
- Whitelist exception: `refHash`, `credential_ref_hash`, `vault_path`, `vault_field` are ALWAYS allowed (documented in the audit-schema whitelist table below).

**Audit whitelist table** (data table required by TASK-005b AC #2):

| Event type | Allowed fields |
|---|---|
| `connect.test` | `type, host, transport, transport_reason, outcome, latency_ms, credential_ref_hash, occurred_at, error` |
| `autofix.propose` | `type, proposal_id, observation_id, target_host, action_class, ladder_level, occurred_at` |
| `autofix.dry-run` | `type, proposal_id, gate_outcome, occurred_at` |
| `action.cancelled` | `type, action_id, reason, occurred_at` |
| `SECRET_LEAK_DETECTED` | `type, field, at` |

Any event with a key outside its whitelist row → key is dropped (not the event) UNLESS the key is in `REDACTED_KEYS` (then also emit the marker).

## 3. Error Handling

All error classes live in `src/secrets/errors.ts` (Vault + secret family) and `src/config/errors.ts` (config family). Every class implements `HomelabError` (see §2.7). The dispatcher in `src/cli/index.ts` MUST:

1. Catch `HomelabError`, write `redact(err.message)` to stderr, exit with `err.exit`.
2. Catch any other error, write generic `unexpected error: {name}` to stderr, exit 1.

### 3.1 Error taxonomy (per TDD §6, restated)

| Class | code | Exit | User message template |
|---|---|---|---|
| `ConfigInvalidError` | `CONFIG_INVALID` | 11 | `config error at {path}: {details}` |
| `ConfigNotFoundError` | `CONFIG_NOT_FOUND` | 12 | `config not found: {path}. Run 'homelab config init' or set --config.` |
| `VaultUnreachableError` | `VAULT_UNREACHABLE` | 20 | `vault unreachable: {address} via {auth_method}` |
| `VaultAuthError` | `VAULT_AUTH_FAILED` | 21 | `vault auth failed: {auth_method}; check {role_id_env}/{secret_id_env}` |
| `VaultPermissionError` | `VAULT_PERMISSION` | 22 | `vault permission denied: cannot read {vault_path}` |
| `SecretMissingError` | `SECRET_MISSING` | 23 | `vault path {vault_path} missing field {vault_field}` |
| `SecretLeakDetectedError` | `SECRET_LEAK_DETECTED` | 24 | `internal safety: attempted to write secret material to audit` |
| `PlatformMismatchError` | `DISCOVERY_PLATFORM_MISMATCH` | 30 | `expected {declared} at {host}, observed {actual}` |
| `TransportFailureError` | `CONNECT_FAIL` | 31 | `{host}: mcp={mcp_result} ssh={ssh_result}` |
| `TypedConfirmMissingError` | (existing) | 40 | (existing) |
| `DelayNotElapsedError` | (existing) | 41 | (existing) |
| `SecretsInBuildError` | `SECRETS_IN_BUILD` | 50 | `secret material detected in dist/; refusing to package` |
| `MutationBarrierError` | `MUTATION_BARRIER_BLOCKED` | 42 | `dry-run blocked mutation: {attemptedMethod}` |

**Recovery strategy notes**:
- 11, 12 → operator action (fix config or supply path). No retry.
- 20, 22 → operator action (Vault reachability / policy). No auto-retry.
- 21 → operator action (rotate role_id / secret_id). No auto-retry.
- 23 → operator action (fix Vault path). No retry.
- 24 → fatal internal bug — surface, do not swallow, do not degrade to logging plaintext.
- 30, 31 → surface in output table; command exit reflects overall pass/fail.
- 40, 41, 42 → these MUST be treated by callers as "expected" outcomes in dry-run reporting.
- 50 → build fails; developer action required.

### 3.2 Redaction

Every error string flowing to stderr, audit, or evidence files passes through `redact()`. `redact()` MUST strip:
- Any 32+ char hex run (probable token).
- Any run matching `hvs\.[A-Za-z0-9._-]+` (Vault tokens).
- Any run matching `-----BEGIN [A-Z ]+PRIVATE KEY-----[\s\S]+?-----END [A-Z ]+PRIVATE KEY-----`.
- Any run matching `AKIA[0-9A-Z]{16}`, `xoxb-[A-Za-z0-9-]+`.

Replacement token: `<redacted>`.
